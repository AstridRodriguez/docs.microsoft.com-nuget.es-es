# <a name="creating-native-packages"></a><span data-ttu-id="bdb42-103">Crear paquetes nativos</span><span class="sxs-lookup"><span data-stu-id="bdb42-103">Creating native packages</span></span>

<span data-ttu-id="bdb42-104">Un paquete nativo contiene código de C++ nativo en lugar de código administrado, lo que permite usarlo en proyectos de C++</span><span class="sxs-lookup"><span data-stu-id="bdb42-104">A native package contains native C++ code instead of managed code, allowing it to be used within C++ projects.</span></span> <span data-ttu-id="bdb42-105">(vea [Native C++ Packages](../consume-packages/finding-and-choosing-packages.md#native-c-packages) [Paquetes nativos de C++] en la sección Consume [Consumir]).</span><span class="sxs-lookup"><span data-stu-id="bdb42-105">(See [Native C++ Packages](../consume-packages/finding-and-choosing-packages.md#native-c-packages) in the Consume section.)</span></span>

<span data-ttu-id="bdb42-106">Para poder usarse en un proyecto de C++, un paquete debe tener como destino la plataforma `native`.</span><span class="sxs-lookup"><span data-stu-id="bdb42-106">To be consumable in a C++ project, a package must target the `native` framework.</span></span> <span data-ttu-id="bdb42-107">Actualmente no hay ningún número de versión asociado a esta plataforma, ya que NuGet trata igual a todos los proyectos de C++.</span><span class="sxs-lookup"><span data-stu-id="bdb42-107">At present there are not any version numbers associated with this framework as NuGet treats all C++ projects the same.</span></span>

> [!Note]
> <span data-ttu-id="bdb42-108">No olvide incluir *native* en la sección `<tags>` del archivo `.nuspec` para que otros desarrolladores puedan buscar el paquete mediante esa etiqueta.</span><span class="sxs-lookup"><span data-stu-id="bdb42-108">Be sure to include *native* in the `<tags>` section of your `.nuspec` to help other developers find your package by searching on that tag.</span></span>

<span data-ttu-id="bdb42-109">Los paquetes nativos de NuGet que tienen como destino `native` proporcionan archivos en las carpetas `\build`, `\content` y `\tools`. En este caso no se usa `\lib` (NuGet no puede agregar referencias directamente a un proyecto de C++).</span><span class="sxs-lookup"><span data-stu-id="bdb42-109">Native NuGet packages targeting `native` then provide files in `\build`, `\content`, and `\tools` folders; `\lib` is not used in this case (NuGet cannot directly add references to a C++ project).</span></span> <span data-ttu-id="bdb42-110">Un paquete también puede incluir archivos de destinos y propiedades en `\build` que NuGet importará automáticamente a los proyectos que consuman el paquete.</span><span class="sxs-lookup"><span data-stu-id="bdb42-110">A package may also include targets and props files in `\build` that NuGet will automatically import into projects that consume the package.</span></span> <span data-ttu-id="bdb42-111">Esos archivos deben tener el mismo nombre que el identificador del paquete con las extensiones `.targets` o `.props`.</span><span class="sxs-lookup"><span data-stu-id="bdb42-111">Those files must be named the same as the package ID with the `.targets` and/or `.props` extensions.</span></span> <span data-ttu-id="bdb42-112">Por ejemplo, el paquete [cpprestsdk](https://nuget.org/packages/cpprestsdk/) incluye un archivo `cpprestsdk.targets` en su carpeta `\build`.</span><span class="sxs-lookup"><span data-stu-id="bdb42-112">For example, the [cpprestsdk](https://nuget.org/packages/cpprestsdk/) package includes a `cpprestsdk.targets` file in its `\build` folder.</span></span>

<span data-ttu-id="bdb42-113">La carpeta `\build` se puede usar para todos los paquetes de NuGet, no solo para los paquetes nativos.</span><span class="sxs-lookup"><span data-stu-id="bdb42-113">The `\build` folder can be used for all NuGet packages and not just native packages.</span></span> <span data-ttu-id="bdb42-114">La carpeta `\build` respeta las plataformas de destino igual que las carpetas `\content`, `\lib` y `\tools`.</span><span class="sxs-lookup"><span data-stu-id="bdb42-114">The `\build` folder respects target frameworks just like the `\content`, `\lib`, and `\tools` folders.</span></span> <span data-ttu-id="bdb42-115">Esto significa que puede crear una carpeta `\build\net40` y una carpeta `\build\net45`, y NuGet importará al proyecto los archivos de propiedades y destinos adecuados</span><span class="sxs-lookup"><span data-stu-id="bdb42-115">This means you can create a `\build\net40` folder and a `\build\net45` folder and NuGet will import the appropriate props and targets files into the project.</span></span> <span data-ttu-id="bdb42-116">(no es necesario usar scripts de PowerShell para importar los destinos de MSBuild).</span><span class="sxs-lookup"><span data-stu-id="bdb42-116">(Use of PowerShell scripts to import MSBuild targets is not needed.)</span></span>
